# Skill Card: 主动学习文本标注
# Active Learning for Text Annotation

**理论基础**: 主动学习 (Active Learning) + 不确定性采样 (Uncertainty Sampling) + 迭代标注优化
**适用领域**: 文本分类数据标注、标签体系冷启动、新标签训练数据生产、多语言标注

---

## ① 算法原理

### 核心思想

标注数据是监督学习的瓶颈。主动学习的核心洞察是：**并非所有样本对模型训练的价值相同**——与其随机抽取样本送人工标注，不如让模型自己判断"哪些样本最困惑"，只把高价值样本送人工。通过迭代循环，用最少的人工标注获得最大的模型性能提升。

### 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                    初始化阶段                                 │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐            │
│  │ 未标注池    │  │ 种子标签    │  │ 标注器     │            │
│  │  (大量)    │  │ (少量人工)  │  │ (规则/ML)  │            │
│  └────────────┘  └────────────┘  └────────────┘            │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│              主动学习循环 (迭代)                              │
│                                                             │
│   ┌──────────┐     ┌──────────┐     ┌──────────┐          │
│   │ 模型预测  │     │ 不确定性  │     │ 人工验证  │          │
│   │ 全部样本  │  →  │ 采样筛选  │  →  │ 高价值样本│          │
│   │          │     │ (Top-K)  │     │ (纠正偏差)│          │
│   └──────────┘     └──────────┘     └──────────┘          │
│        ↑                                   │                │
│        └────────── 模型重训 ←──────────────┘                │
│                                                             │
│   停止条件: 准确率 plateau / 预算耗尽 / 全部标注完成         │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                    输出: 高质量标注数据集                      │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐            │
│  │ 自动标注   │  │ 人工验证   │  │ 置信度分数  │            │
│  │ (高置信度)  │  │ (低置信度)  │  │ (每样本)   │            │
│  └────────────┘  └────────────┘  └────────────┘            │
└─────────────────────────────────────────────────────────────┘
```

### 不确定性采样策略

主动学习的核心是选择"最值得人工验证"的样本。三种经典策略：

**1. 置信度不确定性 (Least Confidence)**

$$U_{LC}(x) = 1 - \max_{y} P(y|x)$$

模型对样本 $x$ 最可能标签的置信度越低，不确定性越高。

**2. 边缘采样 (Margin Sampling)**

$$U_{M}(x) = 1 - \bigl(P(y_1^*|x) - P(y_2^*|x)\bigr)$$

两个最可能标签的概率差距越小（越"纠结"），$U_M$ 越大。

**3. 熵不确定性 (Entropy)**

$$U_{E}(x) = -\sum_{y} P(y|x) \log P(y|x)$$

信息熵衡量整个预测分布的不确定性，适用于标签数 > 2 的场景。

### 标注器设计

本技能提供两层标注器：

1. **规则标注器**: 基于关键词匹配（零成本，适合冷启动）
2. **LLM 标注器**: 基于大语言模型（高准确率，适合精细标注）

两种标注器都支持置信度输出，低置信度样本自动进入人工验证队列。

### 关键假设

1. **标注器能力**: 初始标注器（规则或 LLM）能达到 70%+ 的准确率
2. **标签空间封闭**: 标注前标签体系已确定
3. **种子样本质量**: 初始少量人工标注样本必须准确
4. **数据分布稳定**: 标注阶段与推理阶段分布一致
5. **人工验证成本 > 自动标注成本**: 主动学习才有价值

## ② 母婴出海应用案例

### 场景1：新标签冷启动快速标注

**业务问题**

标签体系发现新痛点"腰贴 adhesive 过敏"，需要至少 100 条标注样本训练分类器。传统外包标注每条 0.8 美元，100 条 = 80 美元，需 2-3 天交付。

**数据要求**

| 数据 | 说明 | 数量 |
|------|------|------|
| 历史评论库 | 包含潜在"过敏"相关评论的未标注数据 | 5000+ 条 |
| 种子样本 | 人工确认的"腰贴过敏"样本 | 5-10 条 |
| 负样本 | 人工确认的非"过敏"样本 | 5-10 条 |

**主动学习流程**

1. **初始化**: 用 5 条正例 + 5 条负例训练初始分类器
2. **第一轮预测**: 分类器对 5000 条评论预测，输出概率分布
3. **不确定性采样**: 选出 20 条置信度最低的样本
4. **人工验证**: 运营人员验证这 20 条
5. **模型重训**: 加入验证样本重新训练
6. **迭代**: 重复步骤 2-5，共 3-5 轮
7. **终止**: 人工累计验证约 50 条，获得 500 条高质量标注数据

**预期产出**

- 500 条带置信度分数的标注数据
- 其中约 450 条来自自动标注（高置信度），50 条经过人工验证
- 整体标注准确率 ≈ 90-93%

**业务价值**

- 成本: 50 条人工 × 0.8 美元 ≈ 40 美元 vs 传统 500 条 × 0.8 = 400 美元，**成本降低 90%**
- 时间: 2 小时 vs 2-3 天，**时间缩短 90%**

### 场景2：多语言评论批量标注

**业务问题**

母婴出海商家在 Amazon US（英语）、Amazon DE（德语）、乐天日本（日语）同时销售，需将各语言评论统一标注到中文标签体系。

**主动学习多语言适配**

1. **语言无关策略**: LLM 标注器可直接处理多语言文本
2. **统一标签空间**: 标签名称保持中文
3. **按语言独立采样**: 每种语言维护独立的标注池
4. **跨语言质检**: 各语言置信度最低的样本统一抽检

**业务价值**

- 多语言标注人力从 3 名全职降至 0.5 名质检，**人力成本降低 83%**
