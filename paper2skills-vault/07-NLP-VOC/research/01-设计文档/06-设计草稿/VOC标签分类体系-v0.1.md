# VOC 标签分类体系 v0.1

> 适用场景：母婴非食品类跨境电商 VOC 智能洞察
> 设计原则：标准驱动、多维度、多层级、可计算
> 关联文档：《顶层驱动逆向语义建模-VOC路径深度评估报告》

---

## 一、设计原则

| 原则 | 说明 |
|------|------|
| **标准先行** | 标签定义必须锚定标准指标体系，避免多头建模 |
| **MECE 完整** | 同级分类互斥穷尽，边界清晰 |
| **语义本体化** | 标签是跨语言的语义节点，不绑定具体文本表达 |
| **可计算性** | 每个标签有明确的业务定义、数据来源、计算逻辑 |
| **层级弹性** | 支持从粗粒度到细粒度的钻取与汇总 |

### 标签类型说明

| 类型 | 定义 | 示例 |
|------|------|------|
| **基础标签** | 直接从单条 VOC 内容提取的标签 | "尺码偏小"、"物流破损" |
| **聚合标签** | 基于基础标签统计聚合生成的标签 | "尺码问题率"、"物流破损率" |
| **衍生标签** | 多维度标签组合计算生成的标签 | "退货高风险"、"品牌忠诚" |
| **预测标签** | 基于模型预测的用户/产品标签 | "流失倾向"、"爆款潜力" |

### 产品维度标签计算规范（示例）

| 标签 | 业务定义 | 数据来源 | 计算逻辑 | 阈值 |
|------|----------|----------|----------|------|
| 材质过敏 | 用户反馈产品材质导致宝宝皮肤过敏 | 电商评论/工单 | NLP识别"过敏"+"皮疹"+"皮肤"+否定词过滤 | 置信度>0.8 |
| 尺码偏小 | 用户明确反馈尺码比预期小 | 全渠道 | NLP识别"偏小"、"太小"、"tight"等 + 尺码词共现 | 置信度>0.75 |
| 包装破损 | 用户收到时包装已损坏 | 电商评论/工单 | NLP识别"包装"+"破"、"damaged"+"package" | 置信度>0.8 |

### 物流维度特殊说明

**跨境电商物流的特殊节点**：

```
订单下单
    ↓
国内仓备货/海外仓备货
    ↓
打包出库
    ↓
国际运输（空运/海运）
    ↓
目的国清关 ←── VOC热点：清关延误、关税
    ↓
末端派送 ←── VOC热点：时效、态度
    ↓
签收
```

**标签归因逻辑**：当用户说"物流慢"时，需结合订单链路数据判断慢在哪个环节，而非笼统标记"物流慢"。

### 体验维度特殊标注

**代理决策者语义区分**：

| VOC 原文 | 情感主体 | 实际标签 | 说明 |

│   ├── 6.2.2 有害物质 (Toxic Substances)

    ├── 6.4.1 误用风险 (Misuse Risk)
    │   └── 标签：说明书不清 / 使用方式误导
    └── 6.4.2 年龄适配 (Age Appropriateness)
        └── 标签：年龄标识不清 / 超龄使用风险 / 发育阶段不适配
```

### 安全维度处理机制

**安全 VOC 不走常规流程，触发以下机制**：

```
安全标签触发 → 即时预警（< 15分钟）→ 人工复核 → 产品下架评估
                    ↓
              自动关联同批次/同供应商
                    ↓
              48小时内完成风险评估报告
```

| 安全等级 | 触发条件 | 响应时效 | 责任人 |
|----------|----------|----------|--------|
| P0-紧急 | 人身伤害报告（窒息/过敏休克等） | < 15分钟 | 产品安全团队+法务合规部 |
| P1-高危 | 批量安全隐患反馈（≥3例同问题） | < 2小时 | 品控+供应商管理 |
| P2-关注 | 个别安全疑虑（如气味异常） | < 24小时 | 客服+产品团队 |

│       └── 标签：服务要求高 / 自助型 / 无所谓型
│
├── 7.3 情感倾向标签 (Sentiment Profile)
│   ├── 7.3.1 整体倾向 (Overall Tendency)
│   │   └── 标签：积极型 / 中性客观型 / 挑剔型
│   ├── 7.3.2 情感稳定性 (Stability)
│   │   └── 标签：情感一致 / 波动大 / 极端化
│   └── 7.3.3 情感表达方式 (Expression Style)
│       └── 标签：理性分析型 / 情绪宣泄型 / 建设性反馈型
│
└── 7.4 价值标签 (Value Tags)
    ├── 7.4.1 用户价值 (Customer Value)
    │   └── 标签：高价值用户 / 潜力用户 / 低价值用户
    ├── 7.4.2 流失风险 (Churn Risk)
    │   └── 标签：流失高危 / 流失预警 / 稳定 / 忠诚
    ├── 7.4.3 推荐价值 (Referral Value)
    │   └── 标签：KOC潜力 / 口碑传播者 / 负面传播者 / 低调用户
    └── 7.4.4 产品反馈价值 (Feedback Value)
        └── 标签：产品洞察者 / 普通用户 / 无效反馈者
```

## 十、跨维度组合标签

> 单一维度标签往往不足以支撑业务决策，以下为高价值的跨维度组合标签

### 10.1 问题归因组合标签

| 组合标签 | 组成标签 | 业务含义 | 应用场景 |
|----------|----------|----------|----------|
| 供应链质量缺陷 | 产品→质量→材质粗糙 + 产品→质量→做工问题 + 多用户同SKU反馈 | 系统性品控问题 | 供应商考核、批次追溯 |
| 物流链路破损 | 物流→状态→外包装破损 + 物流→状态→产品挤压变形 + 同仓库高频 | 特定仓库/物流商问题 | 物流商更换评估 |
| 尺码系统性偏差 | 产品→尺码→尺码偏小 + 多SKU同向偏差 + 跨用户高频 | 尺码标准设计问题 | 尺码表修订 |
| 安全批次事件 | 安全→化学安全→甲醛超标 + 同批次多例 + 时间集中 | 批次性安全问题 | 紧急下架、召回 |

### 10.2 用户洞察组合标签

| 组合标签 | 组成标签 | 业务含义 | 应用场景 |
|----------|----------|----------|----------|
| 品质敏感流失户 | 用户→品质敏感 + 用户→流失高危 + 产品→质量→负面 | 因品质问题即将流失 | 挽回营销、专属客服 |
| 物流抱怨传播者 | 用户→物流敏感 + 用户→社媒发声型 + 物流→负面 | 在社媒上抱怨物流 | 舆情监控、主动安抚 |
| 高价值忠诚粉 | 用户→高价值用户 + 用户→忠诚 + 体验→品牌认同 | 核心资产用户 | VIP运营、新品试用 |
| 产品共创者 | 用户→产品洞察者 + 体验→建设性反馈 + 反馈深度高 | 高价值反馈源 | 产品调研邀请 |

### 10.3 产品运营组合标签

| 组合标签 | 组成标签 | 业务含义 | 应用场景 |
|----------|----------|----------|----------|
| 爆款潜力款 | 产品→设计→创新实用 + 体验→超预期 + 用户→复购意愿高 | 有望成为爆款 | 加大备货、推广资源 |
| 隐藏缺陷款 | 产品→质量→负面占比低但趋势上升 + 安全→P2关注 | 潜在问题产品 | 加强监控、主动排查 |
| 价格敏感款 | 价格→价格过高高频 + 价格→性价比低 + 销量下滑 | 定价可能过高 | 促销测试、调价 |

## 十一、标签优先级矩阵

> 不是所有标签都需要同等级别的计算精度和更新频率

| 优先级 | 标签类型 | 计算精度要求 | 更新频率 | 说明 |
|--------|----------|-------------|----------|------|
| P0 | 安全维度全部标签 | > 95% | 实时 | 人身安全问题，宁可误报不可漏报 |
| P1 | 产品→质量、物流→时效/状态、用户→流失风险 | > 85% | 每日 | 核心业务指标，直接影响决策 |
| P2 | 产品→尺码/包装、售后→解决效果、体验→宝宝反应 | > 80% | 每周 | 重要但容忍一定误差 |
| P3 | 价格维度、体验→外观、用户→情感倾向 | > 75% | 每月 | 洞察辅助，允许迭代优化 |
| P4 | 体验→情感价值、用户→渠道偏好等 | > 70% | 每季度 | 长期用户研究，精度要求最低 |
