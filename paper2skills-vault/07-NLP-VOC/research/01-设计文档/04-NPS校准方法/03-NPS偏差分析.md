# NPS 样本不均衡偏差分析与解决方案

## 一、问题诊断

### 1.1 各数据源情绪分布差异（关键发现）

| 数据源 | 总样本 | 负面情绪 | 正面情绪 | 情绪偏差 |
|--------|--------|---------|---------|---------|
| Amazon（竞品） | 200,000 | 23.5% | 76.5% | 略偏正 |
| Amazon（momcozy） | 65,595 | 25.0% | 75.0% | 略偏正 |
| Trustpilot | 100,000 | 20.5% | 79.5% | 偏正 |
| Reddit | 6,031 | 51.8%（差评）| 18.3%（好评）| **严重偏负** |
| Zendesk | ~48k | ~100% | ~0% | **极端偏负** |

**核心问题**：不同数据源的情绪基线差异巨大。如果 Proxy NPS 直接按 `promoters% - detractors%` 计算，数据源构成不同会导致 NPS 不可比。

### 1.2 品类级情绪差异

momcozy Amazon 数据中，不同品类的星级分布差异显著：

| 品类 | 1-2星率 | 5星率 | 偏差方向 |
|------|---------|-------|---------|
| 吸奶器内衣 | 25.8% | 45.1% | 偏负 |
| 暖奶器 | 30.3% | 49.6% | 偏负 |
| 一体穿戴式吸奶器 | 26.7% | 55.0% | 中等 |
| 哺乳内衣 | 14.4% | 62.3% | 偏正 |
| 消毒器 | 7.5% | 79.5% | **严重偏正** |

### 1.3 偏差产生机制

```
NPS_observed = NPS_true + Bias_source + Bias_category + Bias_brand

其中：
- Bias_source: 数据源本身的情绪倾向（Reddit 偏负，Zendesk 极端负）
- Bias_category: 品类的天然情绪分布（消毒器好评率高，暖奶器投诉多）
- Bias_brand: 品牌在不同数据源/品类的分布差异
```

**场景示例**：
- 品牌 A 在 Reddit 上声量高 → NPS 被低估
- 品牌 B 在 Zendesk 上工单多 → NPS 严重低估
- 品牌 C 主营消毒器 → NPS 被高估（消毒器本身好评率高）

## 二、解决方案候选

### 方案 A：数据源分层校准（推荐）

**思路**：为每个数据源计算情绪基线，将原始 NPS 校准到统一基准。

```python
def calibrate_nps(nps_observed, source_baseline, global_baseline):
    """
    nps_observed: 该数据源观测到的 NPS
    source_baseline: 该数据源的全局情绪基线（如 Reddit detractor 率）
    global_baseline: 跨数据源的全局基线
    """
    return nps_observed - (source_baseline - global_baseline)
```

**实施步骤**：
1. 计算每个数据源的情绪基线（promoter/detractor/neutral 比例）
2. 设定全局基准（如以 Amazon 为基准）
3. 每个数据源的 NPS 按基线差校准
4. 跨源对比时使用校准后 NPS

**优点**：不改变原始数据，校准逻辑透明
**缺点**：需要足够大的样本量来估计基线

### 方案 B：Propensity Score 加权

**思路**：为每条评论赋予权重，使加权后的样本在关键维度（数据源、品类、语言）上与目标分布一致。

```python
# 目标分布：各数据源等权，或按实际市场份额加权
target_distribution = {
    "amazon": 0.40,
    "trustpilot": 0.25,
    "reddit": 0.15,
    "zendesk": 0.20,
}

# 计算每条评论的权重
weight = target_distribution[source] / actual_distribution[source]
```

**优点**：可直接用于任何下游指标计算
**缺点**：权重极端时（如 Zendesk 权重很高）会放大噪声

### 方案 C：品类标准化 NPS

**思路**：NPS 不在全局比较，而是在品类内比较。

```python
def category_relative_nps(brand_nps, category_avg_nps):
    """品牌 NPS 相对于品类均值的偏离"""
    return brand_nps - category_avg_nps
```

**优点**：消除了品类天然差异
**缺点**：无法比较跨品类表现；需要品类有足够多样本

### 方案 D：情绪感知采样（预防性）

**思路**：在数据采集阶段就控制情绪比例，而非事后校准。

```python
# 目标情绪比例
target_sentiment = {"promoter": 0.5, "passive": 0.3, "detractor": 0.2}

# 每层按目标比例采样
for source in sources:
    for sentiment in ["promoter", "passive", "detractor"]:
        n_target = total_target * target_sentiment[sentiment]
        actual = df[(df.source == source) & (df.sentiment == sentiment)]
        # 过采样或欠采样至目标比例
```

**优点**：从根源解决不均衡
**缺点**：需要可控的采样流程；可能丢失真实分布信息

### 方案 E：综合校准框架（最终推荐）

结合 A + C + 数据源标记：

```
校准 NPS = 原始 NPS
           - 数据源基线偏差（方案 A）
           - 品类基线偏差（方案 C）
           + 品牌效应（保留的真实差异）
```

同时保留原始 NPS 作为对比，在报告中标注数据源构成。

## 三、关键决策点（需与用户对齐）

### 决策 1：Zendesk 工单数据如何处理？

Zendesk 当前数据星级评分全为 0，负面情绪 100%。

| 选项 | 处理方式 | 对 NPS 的影响 |
|------|---------|-------------|
| A | 排除 Zendesk，仅用于客诉分析 | NPS 仅基于评论数据 |
| B | Zendesk 单独计算「客诉满意度」，不纳入 NPS | NPS 与客诉指标分离 |
| C | Zendesk 纳入但高权重校准 | 需要设计校准系数 |
| D | Zendesk 也打星级（基于文本情绪推断）| 需要额外的情绪模型 |

**建议**：选 B。Zendesk 工单天然偏负，与评论 NPS 是不同概念。类比：NPS 是「满意度」，Zendesk 是「客诉解决率」。

### 决策 2：Reddit 数据是否纳入 NPS？

Reddit 差评率 51.8%，远高于其他数据源。

| 选项 | 处理方式 |
|------|---------|
| A | 纳入，但权重降低（如 0.3x） |
| B | 纳入，数据源校准后使用 |
| C | Reddit 单独作为「社区口碑指数」，不纳入 NPS |
| D | 排除 Reddit |

**建议**：选 C。Reddit 是讨论社区，情绪表达更极端，适合独立分析。

### 决策 3：校准基准如何选择？

| 选项 | 基准 | 适用场景 |
|------|------|---------|
| A | Amazon 为基准（最中性，样本最大） | 日常监控 |
| B | 跨源等权平均 | 跨品牌对比 |
| C | 行业基准（第三方数据） | 与行业对标 |
| D | 不校准，原始 NPS + 数据源构成披露 | 透明度优先 |

**建议**：选 A + D。以 Amazon 为基准进行校准，但报告中同时披露原始 NPS 和数据源构成。

### 决策 4：品类标准化是否必要？

| 选项 | 处理方式 |
|------|---------|
| A | 全局 NPS 用于品牌整体对比 |
| B | 品类内 NPS 用于精细化分析 |
| C | 同时提供全局 + 品类标准化两套指标 |

**建议**：选 C。全局 NPS 给管理层看整体，品类 NPS 给产品团队看细节。

## 五、momcozy 数据的特殊考量

momcozy 当前数据特征：
- 全部来自 Amazon 商品评论（非工单）
- 负面情绪 25.0%，与竞品 Amazon 数据（23.5%）非常接近
- 这意味着 **momcozy 在 Amazon 上的情绪表现与竞品基本一致**

但需要注意：
1. **品类结构差异**：momcozy 主营吸奶器/哺乳内衣（偏负），竞品可能品类分布不同
2. **语言分布差异**：momcozy 非英语评论占 33%，竞品可能英语占比更高
3. **时间差异**：momcozy 数据是 2504-2604（最近一年），竞品数据时间跨度可能不同

建议 momcozy NPS 计算时：
- 在 Amazon 评论范围内与竞品直接对比（基线一致）
- 若要纳入 Zendesk/Reddit，必须走校准流程
