FIG.7 User Interaction Sequence — Dept Polarity Filter Playwright 实测（产品中心/品线 dashboard）
User
Browser
Superset
voc_bi
Redis
  1. [1]
    User → Browser: 打开 /dashboard/3/
  2. [2]
    Browser → Superset: GET /dashboard/3/
  3. [3]
    Superset → voc_bi: SELECT * FROM v_dept_topic_summary WHERE dept_owner='产品中心/品线' ORDER BY hit_count DESC LIMIT 10
  4. [4]
    Superset → Redis: 缓存查询结果（TTL 5min）
  5. [5]
    Browser ← Superset: HTML + chart-data API → ECharts 渲染 质量感知 20.2k / 易用性 16.4k / 延迟 12.3k
  6. [6]
    User → Browser: 选择「情感极性=负向」+ 点击 Apply filters
  7. [7]
    Browser → Superset: POST /api/v1/chart/data + filter_state.value=['负向']
  8. [8]
    Superset → voc_bi: ... WHERE dept_owner='产品中心/品线' AND polarity='负向' ...
  9. [9]
    Browser ← Superset: chart-data JSON → 重渲染 延迟 12.3k / 尺码小 4.75k / 使用体验差 3.44k + "Applied filters (1)" badge
Performance (Playwright 实测): 首次渲染 ~800ms · Filter Apply ~3-5s（含 SQL 查询 + ECharts 重绘）· Cache hit ~150ms
