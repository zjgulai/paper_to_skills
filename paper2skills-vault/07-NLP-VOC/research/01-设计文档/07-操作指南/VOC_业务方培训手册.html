二、5 个典型场景

场景 1：产品经理找产品改进项

1打开 产品中心 dashboard → 滚到下半部分「D-Action 行动队列」表
2看 priority_score 列降序：当前 Top 3 是「延迟 4046.9 / 尺码小 2843.8 / 使用体验差 2183.2」
3点击行 → 看 biz_action 列了解推荐动作 → 看 strategy_pkg 列了解策略包归属（核心体验改良包等）

进阶：还想看跨国家差异？打开 D-Diag-Product 看 P-2 热力图（OTHER 国 44.69% / UK 35.30%）。

场景 2：客服 leader 看负向工单

1打开 全球客服中心 dashboard
2左侧「过滤」面板 → 「情感极性」选「负向」→ 点击 Apply
3看 D-Action 表：客服中心 Top 行动项 + biz_action（如「客服部：升级响应 SLA」）

追问数据源：打开 D-Diag-Service S-3 矩阵看是 trustpilot 还是 zendesk 客户在抱怨。

场景 3：供应链查物流投诉

1打开 仓储物流部 dashboard
2看 D-Action 表 Top 5：「取消-用错卡 1407.5 / 缺件少件 1057 / ...」
3追问明细：打开 D-Diag-Service S-4 仓储物流 SAT 明细排行（SAT_L3_01 物流 36K hits / SAT_L3_03 包装 / SAT_L3_06 配送）

场景 4：品控部找质量缺陷

1打开 品质管理中心 dashboard
2看 D-Action 表 Top 项：「磨损老化 2581 / 功能失效 1804 / 异味过热 938」+ biz_action「品控部：建立缺陷闭环 + 供应商改进」
3追问国家分布：打开 D-Diag-Service S-1 看「品质 US 90.09% / UK 76.24%」

场景 5：老板看一周全部门 Top 20

1打开 D-Action 总览 dashboard
2顶部大表「全部门 Top 20」按 priority_score 排序看跨部门优先级
3下面 7 个部门小表展开看各自 Top 10 → 部门 leader 自取
⚠️ 注意品牌市场中心 4 个 100% 负向条目（Elvie / Tommee Tippee / Spectra / Philips Avent）：这是 竞品声量监测而非自家整改项 — 用户在抱怨这些竞品 vs 我们。建议按品牌监测流程处理而非内部整改。

三、过滤器使用

每个部门 dashboard 左上角都有「过滤」面板，可按情感极性筛选：

  1. 左侧「过滤」面板 → 点击「情感极性」下拉
  2. 选择「负向 / 正向 / 中性」其中之一
  3. 点击底部「Apply filters」按钮
  4. 3-5s 后图表数据会重绘，顶部会显示「Applied filters (1)」badge
  5. 清空：点击「清除所有」

Overview dashboard 1：左侧也有过滤器（数据源 / 产品线 / Proxy NPS），但仅对饼图（数据源分布、NPS 分布）生效。

已知小限制：当前数据是单批 ETL（month=2026-05-01 单点），所以「时间维度过滤」和「趋势线」会显示单点 — 这是预期内的，下个 Phase 增加真实时间戳后会自动恢复。

四、FAQ（8 问）

Q1：D-Action 表的 priority_score 是怎么算的？
priority_score = hit_negative × pct_negative / 100。体量 × 负向占比 双重加权。值越大表示越该优先处理。例如「延迟」hit_negative=7043, pct_negative=57.46% → priority=4046.9。
Q2：为什么有些 biz_action 是空的？
字典 v4.3 里 ~15 个产品中心标签的 biz_action 还是占位符「【待填写】」，已经在 chart 里被过滤掉。如果你想看有哪些 → 联系 BI 团队。
Q3：品牌市场中心怎么有 4 个竞品名（Elvie / Tommee Tippee）也在 Top 20？
这是 竞品监测信号。100% 负向 = 用户在抱怨这些竞品 vs 我方。按品牌监测流程处理，不是自家产品整改项。
Q4：图表加载慢怎么办？
第一次加载需要 5-10s（视图 SQL 全量跑）。第二次会从 Redis 缓存返回 < 2s。如果一直转圈 → 强制刷新（在 URL 末尾加 ?force=true）。
Q5：能否导出数据到 Excel？
每个 chart 右上角的「⋮」菜单 → Export to CSV / Excel。dashboard 整体导出走 Settings → Export Dashboards 拿 ZIP。
Q6：可以分享 dashboard 链接给同事吗？
可以。直接复制 URL 发给对方。但对方需要先登录（admin / voc_admin_2026 临时账号 — Phase 8 会接 SSO，每人单独账号）。
Q7：数据每天/每周更新吗？
当前是单批 ETL（2026-05-14）。下个 Phase 8 会做月度自动 cron。如果你看到的数据 2 周以上没变 → 联系 BI 团队跑 ETL。
Q8：发现 chart 数据不对怎么办？
先点击 chart 右上角「View as table」看明细数据是否符合预期。如果明细对但聚合不对 → 联系 BI 团队（dashboard owner: VOC NLP）。
