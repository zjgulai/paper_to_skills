# VOC 萃取覆盖率深度诊断报告

## 一、数据概览

| 指标 | 数值 |
|------|------|
| VOC 总记录 | 364,569 |
| 有标签 | 152,650 (41.9%) |
| 零标签 | 211,919 (58.1%) |
| 目标覆盖率 | >70% |
| 差距 | -28.1% |

## 二、五大根因分析

### 根因1: Trustpilot 多语言断层（影响 67,271条 / 31.7%）

Trustpilot 74,070条零标签中：
- 53.6% 法语 (39,690条)
- 37.2% 德语 (27,581条)
- 仅 9.2% 英语 (6,799条)

**结论**：萃取引擎仅有英文关键词库，无法匹配法语/德语评论。Trustpilot总量166,889条，英语仅27,801条（16.7%）。

### 根因2: 品线缺失导致无法匹配个性化标签（影响 122,219条 / 57.6%）

品线为 None 的 173,790 条记录中，仅 29.7% 有标签：

| 品线状态 | 记录数 | 覆盖率 |
|----------|--------|--------|
| 有品线 | 190,779 | 53.4% |
| 品线None | 173,790 | 29.7% |

品线为None的按数据源：
- Amazon竞品: 79,871 (46.0%)
- Trustpilot: 75,977 (43.7%)
- Zendesk: 11,828 (6.8%)
- Momcozy: 5,324 (3.1%)
- Reddit: 790 (0.5%)

**结论**：Amazon竞品和Trustpilot缺乏品线推断机制，导致无法过滤到对应的个性化标签。

### 根因3: 标签关键词库不足（影响 89,700条 / 42.3%）

即使品线推断正确，仍有大量记录无法匹配任何标签：

| 品线 | 总数 | 零标签 | 该品线零标签率 |
|------|------|--------|---------------|
| 吸奶器 | 47,594 | 25,094 | 52.7% |
| 内衣服饰 | 56,432 | 24,763 | 43.9% |
| 智能母婴电器 | 41,670 | 17,093 | 41.0% |
| 喂养电器 | 19,004 | 10,129 | 53.3% |
| 家居家纺 | 18,793 | 8,546 | 45.5% |
| 母婴综合护理 | 7,286 | 4,075 | 55.9% |

高频但未覆盖的关键词：pillow, bottle, order, support, great, love, easy, really 等。

### 根因4: Zendesk 客服工单噪音（影响 34,476条 / 16.3%）

Zendesk零标签文本特征：
- 31.8% 含 "order"（订单查询）
- 13.4% 含邮件线程标记（sent from / on ... wrote）
- 11.9% 含邮箱地址
- 5.3% 含 "track"（物流追踪）
- 4.0% <50字符

典型样本：
- "Is there any estimated delivery date?"
- "Hi, I just checked with my bank and the payment was successful processed."
- "This is a follow-up to your previous request #929548"

**结论**：大量Zendesk记录是订单/物流/退款客服对话，不含产品体验信息，本就不应被萃取打标。

### 根因5: 产品主数据映射缺失（影响 16,887条 / 8.0%）

Momcozy自有数据19,808条中：
- 16,887条（85.3%）SPU在产品主数据中无VOC品线映射
- 仅2,921条（14.7%）有完整的品线映射

**结论**：产品主数据本身缺失品线信息，导致自有数据也无法准确推断。

## 三、产品主数据补全效果

### 3.1 补全52个空值英文名

| 操作 | 数量 |
|------|------|
| 补全空值英文名 | 52 |
| 清洗分列（有型号） | 255 |
| 无型号保留原样 | 176 |

### 3.2 口语表达变体词库

生成 255 个 SPU 的 5 种变体类型：
- model_standalone: 255（如 M5, S12）
- brand_model: 255（如 Momcozy M5）
- line_model: 169（如 breast pump M5）
- model_name: 255（如 M5 Wearable Breast Pump）
- model_lowercase: 255（如 m5, s12）

### 3.3 型号匹配对零标签的覆盖效果

在全部 211,919 条零标签记录中：
- 精确型号匹配（排除Air）: 6,442条 (3.04%)
- 其中真正属于Momcozy产品的约 4,000-5,000条

**结论**：型号匹配本身只能直接覆盖约3%的零标签，价值主要在品线推断辅助而非直接打标。

## 四、改进方案建议

### 方案A: Trustpilot 多语言支持（预期提升 +18%）

**措施**：
1. 对法语/德语评论进行语言检测和机器翻译
2. 翻译后送入现有英文萃取引擎打标
3. 或：为高频法语/德语关键词建立多语言标签映射

**预期效果**：Trustpilot英语覆盖率从当前约30%提升至约50%，整体覆盖率提升约 (166,889×0.2)/364,569 ≈ +9%

### 方案B: 品线推断增强（预期提升 +12%）

**措施**：
1. 基于产品主数据补全后的型号词库，建立型号→品线映射
2. 对Amazon竞品数据，通过ASIN+产品描述匹配到Momcozy产品主数据
3. 对无型号文本，使用文本分类模型推断品线（pump/bra/monitor等关键词）

**预期效果**：品线缺失率从47.6%降至20%，释放约80,000条记录的个性化标签匹配能力，整体覆盖率提升约 +12%

### 方案C: Zendesk 工单过滤（预期提升 +5%，非直接提升覆盖率而是减少分母）

**措施**：
1. 建立Zendesk工单分类器，识别非产品体验类工单（订单/物流/退款）
2. 将这些记录标记为"不适用萃取"，从覆盖率计算中排除
3. 或：单独建立客服主题标签体系

**预期效果**：排除约20,000条纯客服工单后，有效覆盖率基数变为344,569，当前有标签152,650占比44.3%，提升约 +2.4%

### 方案D: 标签关键词库扩充（预期提升 +8%）

**措施**：
1. 部署74个ALCHEmist Label Functions到生产环境
2. 基于零标签样本高频词，新增通用标签（如 Order Experience, Customer Service, General Satisfaction 等）
3. 引入LLM辅助打标，对长尾文本进行语义理解

**预期效果**：有品线但零标签的89,700条中覆盖约30,000条，整体覆盖率提升约 +8%

### 综合预期

| 方案 | 预期提升 | 实现难度 | 优先级 |
|------|----------|----------|--------|
| A 多语言支持 | +9% | 中 | P1 |
| B 品线推断增强 | +12% | 中 | P1 |
| C Zendesk过滤 | +2% | 低 | P2 |
| D 标签库扩充 | +8% | 中 | P1 |
| **合计** | **~+31%** | | |

当前41.9% + 31% = 72.9%，可达标。

## 五、下一步行动建议

1. **立即执行**：部署74个ALCHEmist Label Functions，快速覆盖高频场景
2. **短期（1-2周）**：建立Trustpilot法语/德语关键词映射
3. **中期（2-4周）**：增强品线推断（型号匹配+文本分类）
4. **持续优化**：Zendesk工单分类，区分"可萃取"与"不可萃取"记录
