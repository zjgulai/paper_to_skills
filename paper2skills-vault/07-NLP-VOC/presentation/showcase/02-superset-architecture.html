关键决策 1 · 普通 VIEW 替代物化视图

查询 < 2s 即可满足 Superset 渲染；省掉 REFRESH 调度复杂度，filter 切片实时生效。

关键决策 2 · Chart→Dashboard 双 PUT 协议

先 PUT chart 建立 ownership，再 PUT dashboard 写 layout，避免 Superset 「空占位符」陷阱。

关键决策 3 · Filter scope 限定到同 dataset

跨 dataset filter 会被 Superset 静默忽略。Overview 与 Dept 两套 filter 完全隔离。

Speaker Notes · 09 Superset 平台架构

这一页要让管理层抓住三件事：第一，平台不依赖任何 SaaS，自家 Docker 起，一键起服务； 第二，所有 chart / dashboard / filter 都是 REST API 工厂脚本生成，配置入 git， 换台机器 10 分钟就能重建；第三，没有 LLM 成本，纯工程时间约 7 小时完成 Phase 7。 设计决策点抽 3 个就够了：用普通 VIEW 节省运维，2-call 协议保证 Superset 看板正确装配， filter 必须和 dataset 同源否则静默失效。这三点也是 Phase 8 产品化上线时必须延续的工程纪律。
