```

Step 2: VOC萃取（基础层）

Step 3: 【NEW】语义蓝图结构化
├── 产品属性图谱: 从评论提取结构化属性图
│   └── 吸奶器 → {吸力:9档, 噪音:45dB, 材质:硅胶, 便携性:是}
├── VOC语义蓝图: 提取(方面,情感,持有者,原因)四元组
│   ├── (噪音,负面,用户A,"办公室使用怕吵")
│   ├── (吸力,正面,用户B,"奶量充足"),
│   └── (APP,正面,用户C,"智能记录方便")
├── 行为意图树: 从用户行为序列解析意图
│   └── 搜索"静音吸奶器" → 浏览3款 → 比价 → 加入购物车(未购买) → COMPARISON意图
├── 客服对话决策图: 从客服工单提取问题-解决路径
│   └── 噪音投诉 → 诊断:使用场景 → 建议:夜间模式/降噪配件
└── 跨语言语义对齐: 统一多市场产品描述
    └── en:"breast pump" = zh:"吸奶器" = ja:"搾乳器" = de:"Milchpumpe"

Step 4: 画像生成
├── PERSONABOT: 生成"职场背奶妈妈/静音困扰/配件敏感"画像
│   └── 输入增强: VOC语义蓝图四元组 + 行为意图树
└── SoMeR: 编码为多视角嵌入向量
    └── 融合: 文本嵌入 + 属性图嵌入 + 意图树嵌入

Step 5: 决策智能桥接
├── GPLR: 标签化为"职场背奶妈妈 + 静音敏感型"
│   └── 输入增强: 语义蓝图持有者信息提升标签准确度
├── Kano: "静音"=基本型(必须保障), "APP"=魅力型(差异化)
│   └── 输入增强: 属性图谱提供结构化属性-情感关联
├── iReFeed: Q1优先"静音改进", Q2"APP升级", 跳过"彩色外壳"
│   └── 输入增强: 属性层级关系指导需求依赖挖掘
├── 跨语言对齐: 全球统一"静音"需求优先级
│   └── US/UK/日本/德国 → 统一结构对比 → 日本更关注噪音
└── 上市后发现部分用户流失 → TSCAN分析原因 → 离线RL优化触达

Step 6: 策略输出
├── 产品创新: Q1重点优化降噪，APP作为差异化亮点
│   └── 全球版本: 日本市场优先降噪，德国市场优先材质认证
├── 营销策略: "静音敏感型"人群包 → 推送降噪配件预售
│   └── 多语言文案: 跨语言语义对齐保证各市场信息一致
├── 流失挽回: 配件成本敏感用户 → 配件订阅优惠
│   └── 客服引导: 对话决策图指导客服主动推荐降噪方案
└── 用户旅程优化: 行为意图树识别比价阶段 → 及时推送优惠

Step 7: 【NEW】MAS多智能体策略仿真与执行
├── P2 VOC数据分析师: 35万评论 → 7-Agent深度分析
│   └── 输出: "价格"主题32% + "套装组合"18% + "竞品对比"15%
├── P3 MARL动态定价: Monodense弹性(-0.8) + 竞品价格
│   └── US降价15% / UK降价8%+bundle / DE维持价+增值服务
├── P4 多目标推荐: 用户画像"职场背奶妈妈/静音敏感"
│   └── ConversionAgent↑静音款 + ProfitAgent↑高利润套装
│   └── Coordinator权重: Conversion=54.5%, Profit=33.8%
└── P1 行为仿真: 1000 Agent × 7天促销周期验证
    └── Revenue +22% / Incremental 68% / 品牌忠诚度Momcozy+0.12
    └── 回传: 配件库存风险35% → 调整P3定价 → 风险降至12%

预期效果（含语义蓝图层 + MAS层）:
- 产品迭代成功率 +30% → +45%（结构化输入提升决策准确度）
```
