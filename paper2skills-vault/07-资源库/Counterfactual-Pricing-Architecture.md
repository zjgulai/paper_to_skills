# 跨学科「反事实推演 → 动态定价」全链路 AI 决策系统架构方案

> **适用场景**：母婴跨境电商品牌（面临激烈的价格战、毛利被不断压缩、多业务线目标冲突）
> **设计原则**：结构化非共识 · 反事实基线验证 · MAS 跨部门利益制衡
> **版本**：v1.0 · 2026-06-20

---

## 一、破局洞察：为什么传统定价会陷入死胡同？

传统的动态定价（如 Amazon Repricer）通常基于规则（竞品降价我跟进，BSR 下降我降价）或滞后的销量预测。这种模式存在致命缺陷：
1. **陷入内卷（Race to the bottom）**：大家都在看同样的竞品数据，使用同构的算法，最终只能互相绞杀毛利。
2. **无法量化真实增量（Uplift）**：降价多卖出的 100 单中，有多少是原本就会全价购买的客户？这种“蚕食效应（Cannibalization）”在传统预测模型中被完全掩盖。
3. **部门目标割裂**：运营追求单量（降价），财务追求毛利（提价），供应链害怕断货（控价），单维度的定价算法总是造成全局次优。

**本方案的核心范式**：引入因果推断的 **反事实基线（Counterfactual Baseline）**，计算真实的利润 Uplift；并引入 **MAS（多智能体）董事会博弈**，实现跨部门的纳什均衡。

```
L1 异构信号解析层 (The Orthogonal Data Arbitrage)
   - 滞后信号：历史销量、现有价格、Review 评分
   - 领先信号：社媒长尾痛点发帖加速度（Velocity）
   - 正交信号：海关数据推演的竞品库存深度、原材料大宗商品汇率

L2 因果干预推演层 (The Counterfactual What-If Machine) [核心核心]

   - 反事实预测 (Do-Calculus)：P(Y | do(Price=X))
   - 真实增量计算 (Uplift Modeling)：剔除蚕食效应后的净利润增量

   [竞价博弈 Agent] 模拟对手跟随策略（强化学习 DRL）
   [流量投放 Agent] 评估 ACOS 与转化率的匹配度
   [供应链 Agent]   预警库存枯竭周期与补货成本
   [财务总监 Agent] 守住底线净利率
                   ↓ Agent 间基于 LLM 与规则混合的多轮辩论 (Nash Equilibrium)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
L4 决策阻断与执行层 (The Reality Checker Gate)
   - 三轨验证：成本合规、平台规则合规（防被判操纵价格）、品牌调性风控
   - 置信度门控 (Confidence Gating)：
     - 高置信度/低风险 -> 自动下发调价指令至 API
     - 低置信度/高风险 -> 拦截，生成附带反事实推演图表的审批报告
                   ↓ 自动化执行 or 阻断通知
```

## 三、核心工作流：反直觉的“不降价”决策是如何诞生的？

**场景举例**：黑五前夕，主要竞品突然降价 15%。传统算法会立即跟进降价。

1. **正交信号扫描 (L1)**：系统不仅感知到竞品降价，同时从海关数据推断出竞品最近两个月到港货物极少，且原材料（如塑料/硅胶）价格在飙升。
2. **因果推演 (L2)**：
   - 方案 A（跟进降价 15%）：预测单量暴增，但模型发现会触发断货阈值，且利润 Uplift 为负（降价损失 > 增量利润）。
   - 方案 B（维持原价甚至提价 5%）：反事实推演表明，竞品极大概率在 2 周内断货。一旦竞品断货，我们将获得定价权。
3. **MAS 博弈 (L3)**：
   - *竞价 Agent*：“如果不降价，我们的 BSR 本周会跌出前 10！”
   - *供应链 Agent*：“海运严重堵塞，我们如果降价，剩下的库存撑不到圣诞节，空运补货会吃掉所有利润！”
   - *财务 Agent*：“支持方案 B。宁愿损失本周的 30% 销量，博取竞品断货后的 50% 溢价空间。”
4. **最终输出 (L4)**：
   - 决策：**保持原价，切断无效广告，囤积库存。**
   - 商业价值：避开了血腥的绞杀，利用对手的供应链断裂，在旺季后半段实现了 40% 的超额利润。这就是算法寻找**结构性漏洞（Structural Inefficiencies）**的实战。

## 五、实施路线图与陷阱避坑 (The Implementation Path)

### Phase 1：解耦与因果基线建立 (Month 1-2)
- **动作**：不要急着上线自动定价。先用因果推断模型并行跑在历史数据上，计算历史中每一次大促的“真实净利润 Uplift”，向管理层证明“过去的很多降价是无效甚至亏损的”。
- **目标**：建立信任。

### Phase 2：旁路博弈系统上线 (Month 3-4)
- **动作**：将 MAS 董事会博弈模型作为“旁路建议系统”接入。每次人工需要调价时，系统输出一份由各个 Agent 撰写的正反方评估报告（带量化数据）。
- **目标**：修正人类的短视偏见。

### Phase 3：闭环自动化与门控放权 (Month 5-6)
- **动作**：设置严格的门控（Confidence Score > 0.85 且亏损风险 < 5%）。对低风险商品逐步放权给 AI 直接操作 API 定价。
- **目标**：落地执行，实现降维打击。

### ⚠️ 架构陷阱
1. **因果混淆 (Confounding Bias)**：把节假日带来的自然销量增长，归功于降价。这是传统预测最容易犯的错，必须强制使用 `Do-Calculus` 来分离混淆因子。
2. **MAS 沦为“角色扮演”**：如果 财务 Agent 只是 Prompt 设定（“你是一个财务”），那是玩具。它必须真正挂载 FBA 仓储费率计算 API 和利润率硬性阈值，带有“一票否决权（Veto Power）”。
3. **忽略平台封杀规则**：频繁且无底线的高频调价会被 Amazon 判定为价格操纵并暂停购物车（Buy Box），因此 L4 的合规阻断引擎至关重要。
