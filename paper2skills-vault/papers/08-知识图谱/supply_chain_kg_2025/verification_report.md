## 三、代码修复记录

### 问题：PageRank 总和不为 1

**原因**：初版使用列归一化，在稀疏有向图中悬空节点（无出边）导致概率质量流失，PR 总和仅约 0.32。

**修复**：改为**行归一化**（每行=从 src 出发的跳转概率），对悬空节点做均匀分布处理，并使用 `W.T @ PR` 的标准 PageRank 迭代形式。修复后 PR 总和为 1.000 ± 0.001。

## 六、已知局限与后续优化方向

1. **图遍历规模**：当前 BFS 在超大图（万级节点）可能较慢，生产环境建议引入图数据库（Neo4j）或使用 networkx 的 Dijkstra 优化
2. **中心度仅做静态预计算**：图谱动态更新后需重新计算；高频更新场景可改为增量更新
3. **Context Shell 为规则模版**：接入真实 LLM 后，可将 Shell 作为 Prompt 的 Context 部分，由 LLM 生成更丰富的分析文本
4. **违约概率来源**：当前为手动录入，生产环境应从信用评级 API 或历史采购数据自动估算
