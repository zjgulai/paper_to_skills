## 3. 代码结构说明

```
model.py (547 行)
├── Resources          # Agent 资源约束（预算/时间/精力）
├── Memory             # 社会记忆（品牌体验/口碑/历史访问）
├── Agent              # 虚拟消费者，含 Persona + 社交关系
├── MarketingEvent     # 营销事件（折扣/新品/会员制）
├── Venue              # 商业场所（咖啡馆/快餐/家庭餐厅）
├── AgentDecisionEngine # mock LLM 语义判断（评分函数+扰动）
├── MarketingSandbox   # 沙盒仿真引擎（多天循环 + WOM 传播）
├── SimulationAnalyzer # 对比报告生成
├── create_agents()    # 工厂函数（含社交网络初始化）
├── create_venues()    # 工厂函数
├── run_ab_simulation()# 控制/实验对比入口
└── self_test()        # 自测（含 3 个业务断言）
```

## 4. 关键设计决策

| 决策 | 原因 |
|---|---|
| 完全 mock，不调用 LLM API | 可离线运行，成本 $0，CI 可跑；生产替换成本低 |
| 评分函数模拟 LLM 语义推理 | 论文核心是系统架构，决策逻辑可参数化近似 |
| 4 种 Persona 枚举 | 覆盖母婴用户主流行为模式，与 RFM 分群可互映射 |
| WOM 一跳传播 | 简化实现，足以展现涌现性口碑效应，不过度工程化 |
| 控制组对比实验 | 符合论文的 pre/post + A/B 评估框架 |
