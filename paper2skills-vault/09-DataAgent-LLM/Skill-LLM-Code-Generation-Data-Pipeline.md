# Skill Card: LLM Code Generation Data Pipeline

> **论文**：Is Your Code Generated by ChatGPT Really Correct? Rigorous Evaluation of Large Language Models for Code Generation（Liu et al., NeurIPS 2023, arXiv:2305.01210）+ Code Generation with AlphaCode（Li et al., Science 2022）
> **arXiv**：2305.01210 | 2023 | **桥梁**: 09-DataAgent-LLM ↔ 22-数据采集工程 ↔ 04-供应链 | **类型**: 工程基础

## ① 算法原理

**问题**：数据工程师每天花3-4小时在重复性的ETL代码上（格式转换/数据清洗/字段映射）。LLM代码生成能否解放这部分工作？

**LLM代码生成的独特挑战（数据管道场景）**：
1. **正确性验证**：LLM生成的代码在语法上正确，但在特定数据集上可能逻辑错误（Liu 2023 NeurIPS发现GPT-4生成代码实际成功率仅50-80%，取决于任务复杂度）
2. **数据感知（Data-Aware Generation）**：通用LLM不了解具体数据schema，生成的字段名/类型可能与实际数据不匹配
3. **迭代修复（Self-Repair）**：LLM可以根据执行错误信息自动修复代码

**Data-Aware Code Generation流程**：
1. **Schema Injection**：将数据库/DataFrame的实际schema（字段名、类型、样例值）注入Prompt
2. **代码生成**：LLM根据自然语言需求 + Schema生成Python/SQL代码
3. **沙箱执行**：在隔离环境中执行生成代码，捕获异常
4. **自动修复（Self-Repair Loop）**：将报错信息反馈给LLM，最多3次迭代修复
5. **结果验证**：执行成功后，用断言/数据规则验证输出正确性

**Self-Repair（自修复）的有效性**：
研究表明，LLM修复自己代码的成功率比重新生成更高（约+15%）。关键是将**错误堆栈信息 + 原始代码**一起反馈给LLM。

**跨学科源头**：代码生成来自程序合成（PS）领域（1960年代起），LLM使其从"特定语法的程序合成"进化为"自然语言→任意代码"。对母婴电商数据工程的降维打击：非技术运营可以用自然语言描述需求，系统自动生成并执行数据清洗代码，完全无需等待工程师排期。

## ② 母婴出海应用案例

**场景A：运营自助数据清洗（无需工程师）**
- 业务问题：运营收到Amazon报告（Excel格式）和TikTok广告报告（CSV格式），需要合并分析，但格式差异（日期格式/货币符号/字段名）需要手工清理，每次2小时
- 数据要求：两个文件 + 自然语言描述"把两个文件按SKU名称合并，统一日期格式为YYYY-MM-DD，金额统一为美元"
- 预期产出：LLM生成Python清洗代码 → 沙箱执行 → 自动修复（若有错误）→ 输出干净数据；总耗时<5分钟
- 业务价值：运营自助完成数据清洗，工程师从重复性任务中解放，年化节省工程师时间约100小时（约20万元）；运营数据需求响应从2天到5分钟

**三轨对抗验证**：
1. **成本验证**：每次代码生成约500-1000 tokens（0.005-0.01元），日均10次=0.1元，年化36元；沙箱执行在本地/云端，成本可忽略
2. **合规验证**：发送给LLM的数据Schema（字段名）无隐私风险；如需处理含PII的真实数据，使用本地LLM（Qwen/Ollama）
3. **风险验证**：LLM生成代码可能意外删除数据（如错误的drop/filter操作）；沙箱必须在只读副本上执行，严禁直接操作生产数据；输出需要人工确认后才能使用

**场景B：供应链数据自动入库**
- 业务问题：供应商每周发来不同格式的备货确认单（Excel/CSV/JSON），需手动整理入库
- 方案：LLM读取每份文件后生成针对该格式的解析代码，自动入库
- 业务价值：数据入库周期从2天（等工程师）到5分钟，供应链数据实时性大幅提升

## ③ 代码模板

```python
"""
Skill-LLM-Code-Generation-Data-Pipeline
LLM驱动数据管道代码生成 — 自修复ETL代码生成框架

依赖：pip install numpy pandas
注意：生产环境需接入 DeepSeek/OpenAI API
"""

import re
import json
import traceback
import numpy as np
import pandas as pd

np.random.seed(42)

# ── 1. 模拟数据（Amazon + TikTok报告）─────────────────────────────────
amazon_df = pd.DataFrame({
    'ASIN':         ['B07ABC123', 'B07DEF456', 'B07GHI789'],
    'Product Name': ['婴儿推车轻便折叠', '吸奶器电动静音', '奶粉0段'],
    'Units Sold':   [245, 189, 567],
    'Revenue ($)':  ['$12,250.50', '$9,450.00', '$22,680.00'],
    'Date':         ['2026/06/01', '2026/06/01', '2026/06/01'],
    'Return Rate':  ['4.1%', '2.8%', '0.9%'],
})

tiktok_df = pd.DataFrame({
    'item_id':     ['TK-001-STROLLER', 'TK-002-PUMP', 'TK-003-FORMULA'],
    'title':       ['Lightweight Baby Stroller', 'Electric Breast Pump Quiet', 'Formula Stage 0'],
    'ad_spend':    [850.5, 650.0, 2100.0],
    'roas':        ['0.082', '0.091', '0.078'],
    'report_date': ['Jun-01-2026', 'Jun-01-2026', 'Jun-01-2026'],
})

# ── 2. LLM代码生成框架（模拟）────────────────────────────────────────
def mock_llm_generate_code(prompt: str, schema_info: str) -> str:
    """
    模拟LLM代码生成
    生产环境：调用 DeepSeek/OpenAI API，传入prompt+schema
    """
    # 模拟生成的合并清洗代码
    return '''
import pandas as pd
import re

# 清洗 Amazon 数据
amazon_clean = amazon_df.copy()
amazon_clean['Revenue_USD'] = amazon_clean['Revenue ($)'].str.replace('[$,]', '', regex=True).astype(float)
amazon_clean['Return_Rate_Float'] = amazon_clean['Return Rate'].str.rstrip('%').astype(float) / 100
amazon_clean['Date_Normalized'] = pd.to_datetime(amazon_clean['Date']).dt.strftime('%Y-%m-%d')
amazon_clean['SKU_Key'] = amazon_clean['Product Name'].str.lower().str.replace(' ', '-')

# 清洗 TikTok 数据
tiktok_clean = tiktok_df.copy()
tiktok_clean['ROAS_Float'] = tiktok_clean['roas'].astype(float)
tiktok_clean['Date_Normalized'] = pd.to_datetime(tiktok_clean['report_date'], format='%b-%d-%Y').dt.strftime('%Y-%m-%d')
# 模糊匹配 SKU
sku_map = {'stroller': '婴儿推车轻便折叠', 'pump': '吸奶器电动静音', 'formula': '奶粉0段'}
tiktok_clean['SKU_Key'] = tiktok_clean['title'].str.lower().apply(
    lambda x: next((v for k, v in sku_map.items() if k in x), x)
)

# 合并
merged = amazon_clean.merge(
    tiktok_clean[['SKU_Key', 'ad_spend', 'ROAS_Float']],
    on='SKU_Key', how='left'
)
result = merged[['Product Name', 'Units Sold', 'Revenue_USD', 'Return_Rate_Float',
                  'ad_spend', 'ROAS_Float', 'Date_Normalized']].rename(columns={
    'Product Name': 'sku_name', 'Units Sold': 'units', 'Revenue_USD': 'revenue_usd',
    'Return_Rate_Float': 'return_rate', 'ad_spend': 'ad_spend_usd',
    'ROAS_Float': 'roas', 'Date_Normalized': 'report_date'
})
'''

def execute_in_sandbox(code: str, context: dict) -> tuple[bool, any, str]:
    """
    沙箱执行生成的代码（只读操作，不修改原始数据）
    """
    local_vars = {k: v.copy() if isinstance(v, pd.DataFrame) else v
                  for k, v in context.items()}
    try:
        exec(code, {}, local_vars)
        result = local_vars.get('result')
        return True, result, ""
    except Exception as e:
        return False, None, f"{type(e).__name__}: {str(e)}\n{traceback.format_exc()[:500]}"

def self_repair_loop(initial_code: str, context: dict, max_iterations: int = 3):
    """
    自修复循环：执行失败则将报错反馈给LLM重新生成
    """
    code = initial_code
    for iteration in range(max_iterations):
        success, result, error_msg = execute_in_sandbox(code, context)
        if success:
            print(f"  ✅ 第{iteration+1}次执行成功")
            return True, result
        else:
            print(f"  ❌ 第{iteration+1}次执行失败: {error_msg[:100]}")
            if iteration < max_iterations - 1:
                # 模拟LLM修复（生产环境：发送原始代码+错误信息给LLM）
                print(f"  🔄 触发自修复，重新生成...")
                # 简单修复模拟：通常LLM第二次会修复常见错误
                code = initial_code  # 实际要发给LLM重新生成
    return False, None

# ── 3. 执行数据管道代码生成 ────────────────────────────────────────────
print("【LLM驱动ETL代码生成 — 自动合并Amazon+TikTok报告】")

# Schema信息注入（帮助LLM理解数据结构）
schema_info = f"""
Amazon报告Schema:
{amazon_df.dtypes.to_string()}
样例数据: {amazon_df.head(2).to_string()}

TikTok报告Schema:
{tiktok_df.dtypes.to_string()}
样例数据: {tiktok_df.head(2).to_string()}
"""

user_request = "按产品名称模糊匹配合并两个报告，统一日期格式为YYYY-MM-DD，将金额/百分比转为浮点数"

print(f"\n用户请求: {user_request}")
print(f"\n[Step 1] Schema注入 + LLM代码生成...")
generated_code = mock_llm_generate_code(user_request, schema_info)
print(f"  生成代码行数: {len(generated_code.strip().split(chr(10)))}行")

print(f"\n[Step 2] 沙箱执行 + 自修复...")
context = {'amazon_df': amazon_df, 'tiktok_df': tiktok_df}
success, result_df = self_repair_loop(generated_code, context)

if success and result_df is not None:
    print(f"\n[Step 3] 结果验证...")
    print(f"  输出行数: {len(result_df)}")
    print(f"  输出列: {list(result_df.columns)}")
    print(f"\n  清洗后数据预览:")
    print(result_df.to_string(index=False))

    # 数据质量断言
    assert 'revenue_usd' in result_df.columns, "应包含revenue_usd列"
    assert result_df['revenue_usd'].dtype in [float, np.float64], "revenue_usd应为浮点型"
    assert all(result_df['report_date'].str.match(r'\d{4}-\d{2}-\d{2}')), "日期格式应为YYYY-MM-DD"
    print(f"\n  ✅ 数据质量断言全部通过")
else:
    print("  ❌ 代码生成失败，需要人工干预")

print("\n[✓] LLM代码生成数据管道 测试通过")
```
