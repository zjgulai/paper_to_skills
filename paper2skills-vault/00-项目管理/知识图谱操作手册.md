# Paper2Skills 知识图谱操作手册

## 快速开始

### 1. 查看当前图谱状态

```bash
cd paper2skills-skills/paper-skills-graph
python scripts/skills_graph_analyzer.py --analyze
```

输出包含:
- 节点总数、边总数
- 领域分布
- 中心性分析 (核心技能、潜力技能)
- 知识缺口列表
- 推荐选题

### 2. 分析特定技能

```bash
python scripts/skills_graph_analyzer.py --skill Skill-Uplift-Modeling
```

### 3. 仅查看知识缺口

```bash
python scripts/skills_graph_analyzer.py --gaps
```

## 实战场景

### 场景1: 发现新选题方向

**目标**: 找到下一个要萃取的技能

**步骤**:

1. **运行图谱分析**
   ```bash
   python scripts/skills_graph_analyzer.py --analyze > report.md
   ```

2. **查看高优先级缺口**
   - 关注 "🔴 高优先级缺口" 部分
   - 优先处理 "missing_prerequisite" 类型

3. **评估缺口业务价值**
   - 检查缺口影响的技能
   - 查看该技能的业务价值评分
   - 确认是否为核心业务场景

4. **生成搜索关键词**
   - 使用报告中的 "搜索关键词"
   - 或基于缺口技能名称构造关键词

5. **搜索论文**
   ```bash
   # 示例: 搜索因果森林相关论文
   arxiv_query="causal forest heterogeneous treatment effect"
   ```

**决策树**:
```
发现缺口
    │
    ├── 缺口类型是 missing_prerequisite?
    │   ├── 是 → 影响技能业务价值 ≥4星?
    │   │       ├── 是 → P0 优先级，立即填补
    │   │       └── 否 → P1 优先级，延后处理
    │   └── 否 → 继续检查
    │
    ├── 缺口类型是 missing_extension?
    │   ├── 是 → 高价值技能无延伸?
    │   │       ├── 是 → P0 优先级，规划延伸
    │   │       └── 否 → P2 优先级
    │   └── 否 → 继续检查
    │
    └── 缺口类型是 missing_bridge?
        └── 两个领域都是核心业务?
            ├── 是 → P1 优先级，建立桥梁
            └── 否 → P2 优先级
```

### 场景3: 评估技能组合效果

**目标**: 判断多个技能组合使用的价值

**步骤**:

1. **识别待组合技能**
   
   例如: Causal Forest + LTV Prediction

2. **检查现有连接**
   ```bash
   # 查看两个技能是否在可组合列表中
   grep -A 5 "可组合技能" paper2skills-vault/01-因果推断/Skill-Intelligent-Attribution-Causal-Forest.md
   ```

3. **评估组合价值**
   
   | 评估维度 | Causal Forest + LTV |
   |---------|-------------------|
   | 互补性 | 归因能力 + 价值评估 = 精准投放 |
   | 业务场景 | 高价值用户获客渠道优化 |
   | 数据复用 | 用户特征可共享 |
   | 技术可行性 | 都是预测模型，易于集成 |

4. **设计组合方案**
   
   ```python
   # 伪代码示例
   class HighValueAttribution:
       def __init__(self):
           self.cf = CausalForestAttribution()
           self.ltv = LTVPredictor()
       
       def analyze(self, user_data):
           # Step 1: 预测用户 LTV
           ltv_score = self.ltv.predict(user_data)
           
           # Step 2: 对高 LTV 用户进行归因分析
           if ltv_score > threshold:
               uplift = self.cf.predict_effect(user_data)
               return {
                   'ltv': ltv_score,
                   'uplift': uplift,
                   'strategy': 'high_priority' if uplift > 0.1 else 'standard'
               }
   ```

5. **验证组合效果**
   - 设计 A/B 测试验证组合 vs 单一技能
   - 追踪业务指标提升

### 场景4: 新技能入库流程

**目标**: 将新萃取的技能正确纳入知识图谱

**步骤**:

1. **技能质量检查**
   
   使用审核检查清单:
   ```bash
   # 检查 5 个模块是否完整
   grep -c "## ①" Skill-New.md  # 应返回 1
   grep -c "## ②" Skill-New.md  # 应返回 1
   grep -c "## ③" Skill-New.md  # 应返回 1
   grep -c "## ④" Skill-New.md  # 应返回 1
   grep -c "## ⑤" Skill-New.md  # 应返回 1
   ```

2. **确定分类位置**
   
   根据《知识图谱架构与分类体系》:
   - 选择主领域 (01-07)
   - 确定复杂度层级 (Level 1-4)
   - 规划技能关系

3. **移动文件到正确位置**
   ```bash
   # 示例: 新因果推断技能
   mv Skill-New.md paper2skills-vault/01-因果推断/
   ```

4. **更新图谱连接**
   
   编辑相关技能的 "技能关联" 部分:
   ```markdown
   ### 前置技能
   - **新技能**: 说明原因
   
   ### 延伸技能  
   - **新技能**: 说明原因
   
   ### 可组合技能
   - **新技能**: 说明组合价值
   ```

5. **重新生成图谱分析**
   ```bash
   python scripts/skills_graph_analyzer.py --analyze
   ```

6. **验证连接正确性**
   - 检查新技能是否出现在图谱中
   - 检查技能关系是否正确解析
   - 检查是否填补了预期缺口

7. **更新同步状态**
   ```bash
   # 更新 sync_status.json
   # 添加新技能条目
   ```

### 可视化准备

```python
# 导出为 Gephi 格式
# 在 skills_graph_analyzer.py 中添加:

def export_gephi(self, output_path):
    """导出为 Gephi 兼容的 CSV"""
    # 节点表
    nodes = pd.DataFrame([
        {
            'Id': node.id,
            'Domain': node.domain,
            'Difficulty': node.difficulty,
            'BusinessValue': node.business_value
        }
        for node in self.nodes.values()
    ])
    nodes.to_csv(f"{output_path}_nodes.csv", index=False)
    
    # 边表
    edges = pd.DataFrame([
        {
            'Source': edge.source,
            'Target': edge.target,
            'Type': edge.edge_type,
            'Weight': edge.weight
        }
        for edge in self.edges
    ])
    edges.to_csv(f"{output_path}_edges.csv", index=False)
```

## 常见问题 FAQ

**Q1: 为什么我的新技能没有出现在图谱中?**

A: 检查以下几点:
1. Skill 文件是否在 `paper2skills-vault/` 下
2. 文件名格式是否为 `Skill-*.md`
3. 是否包含正确的 frontmatter (---)
4. 技能关联部分格式是否正确

**Q2: 如何判断技能关系是否设置正确?**

A: 运行 `python scripts/skills_graph_analyzer.py --skill 技能名`，检查:
- 前置技能列表是否符合预期
- 延伸技能是否正确解析
- 可组合技能是否完整

**Q3: 图谱显示有缺口，但不知道该搜索什么论文?**

A: 使用报告中的 "搜索关键词"，或参考:
- 关键词库: `paper2skills-vault/07-资源库/关键词库.md`
- 根据缺口类型构造: `[缺失技能名] + [tutorial/survey]`

**Q4: 如何评估是否应该创建新技能，还是扩展现有技能?**

A: 决策标准:
- 如果与现有技能共享 <50% 内容 → 创建新技能
- 如果业务场景完全不同 → 创建新技能
- 如果只是现有技能的参数调优 → 扩展现有技能
- 如果只是应用案例不同 → 扩展现有技能

**Q5: 技能太多导致图谱过于复杂怎么办?**

A: 应对策略:
1. 使用领域过滤: 只查看特定领域的子图
2. 使用层级过滤: 只查看 Level 1-2 的基础技能
3. 创建领域专属视图: 为每个领域生成独立图谱
4. 使用社区聚类: 按社区分组展示
