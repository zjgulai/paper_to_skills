## 四、分类体系实践指南

### 4.1 如何为新技能定位

#### 决策树

```
新技能: X算法
    │
    ├── 1. 确定主领域
    │   ├── 处理因果效应? → 01-因果推断
    │   ├── 处理实验设计? → 02-A/B实验
    │   ├── 处理时序数据? → 03-时间序列
    │   ├── 处理库存物流? → 04-供应链
    │   ├── 处理推荐排序? → 05-推荐系统
    │   ├── 处理用户价值? → 06-增长模型
    │   ├── 处理文本数据? → 07-NLP-VOC
    │   └── 处理知识结构? → 08-知识图谱
    │
    ├── 2. 确定复杂度层级
    │   ├── 依赖≥2个前置技能 → Level 3-4 (Advanced/Expert)
    │   ├── 被≥2个技能依赖 → Level 2-3 (Core/Advanced)
    │   └── 无前置/无延伸 → Level 1 或 Level 4
    │
    ├── 3. 确定技能关系
    │   ├── 需要哪些前置技能? → 列出依赖
    │   ├── 可以延伸哪些技能? → 列出发展路径
    │   └── 可以和谁组合? → 列出协同技能
    │
    └── 4. 验证缺口匹配
        ├── 是否填补已有缺口? → 是=高优先级
        └── 是否创造新缺口? → 是=需规划延伸
```

#### 示例: iReFeed 定位过程

```
1. 领域: 处理产品决策优先级 → 07-NLP-VOC
2. 复杂度: 依赖 Kano + TopicImpact → Level 4 (Bridge/Expert)
3. 关系:
   - 前置: Kano, TopicImpact, BERTopic
   - 延伸: 产品路线图自动化, 季度复盘报告
   - 组合: MAA (优先级 → 行动建议), AGRS (摘要 → 优先级输入)
4. 缺口匹配: 填补 Kano 到产品路线图的断点 → P0 优先级
```

### 4.2 技能关联编写规范

#### 前置技能选择原则

| 原则 | 说明 | 示例 |
|------|------|------|
| 必要性 | 必须掌握才能理解本技能 | Causal Forest → 需要 Uplift |
| 充分性 | 掌握后学习本技能无障碍 | 有 Uplift 基础即可学 Causal Forest |
| 邻近性 | 概念距离不超过2跳 | Uplift (1跳) → Causal Forest |

#### 延伸技能规划原则

```
延伸方向1: 深度延伸 (技术深化)
Uplift Modeling → Causal Forest → [MCF]
           ↓
      Doubly Robust Estimation

延伸方向2: 广度延伸 (场景扩展)
Uplift Modeling → 动态定价应用
           ↓
      推荐去偏应用

延伸方向3: 组合延伸 (跨界融合)
Uplift Modeling + LTV → 高价值用户归因
SoMeR + GPLR → 可解释精准营销
Kano + [iReFeed] → 数据驱动产品路线图
```

#### 可组合技能匹配原则

```
互补型组合: 能力互补，1+1>2
├── Causal Forest + LTV = 精准归因+价值分层
├── VOC + Churn = 情感驱动的流失预警
├── 时序 + 库存 = 预测驱动的补货
└── Kano + iReFeed = 结构化需求+优先级排序

协同型组合: 流程衔接，端到端
├── Uplift → LTV = 归因到运营闭环
├── 实验 → 推荐 = 快速迭代优化
├── 预测 → 库存 = 需求到供应闭环
├── TopicImpact → Kano → iReFeed = VOC到产品闭环
└── SoMeR → GPLR → OfflineRL = 画像到触达闭环
```

#### 每年审查

- [ ] 技能是否仍然符合业务需求

### 5.2 图谱扩展策略

#### 策略1: 缺口驱动扩展

```
发现缺口 → 搜索论文 → 萃取技能 → 填补缺口 → 验证连接

示例:
发现: Kano 无延伸到产品路线图的桥梁
搜索: iReFeed (arXiv:2603.28677)
萃取: iReFeed Skill
填补: 建立 Kano → iReFeed → 路线图 连接
验证: 运行图谱分析确认闭环完整
```

#### 策略2: 业务需求驱动扩展

```
业务需求 → 识别能力缺口 → 映射技术方案 → 萃取/组合技能

示例:
需求: 预测新品上市成功率
缺口: 缺乏上新时机预测能力
方案: Doubly Robust + 时序预测
萃取: 上新时机预测 Skill
组合: 建立与现有技能连接
```

#### 策略3: 技术前沿驱动扩展

```
技术趋势 → 评估业务适用性 → 原型验证 → 萃取技能

示例:
趋势: LLM for Causal Inference
评估: 适合母婴出海客服场景
验证: GPT-4 因果推理实验
萃取: LLM-Causal Skill
```

```
边框: 虚线=规划中, 实线=已完成
```

```
前置依赖: → (实线箭头)
延伸发展: → (虚线箭头)
可组合: ↔ (双向箭头)

粗细: 与权重成正比

标签: 关系类型 (可选)
```
