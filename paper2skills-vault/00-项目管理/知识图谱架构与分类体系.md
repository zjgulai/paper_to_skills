## 二、分类体系设计

### 2.1 三维分类模型

```
                    业务应用场景
                         │
            ┌────────────┼────────────┐
            │            │            │
      因果推断 ──────── 增长模型 ─────── 预测分析
            │            │            │
            └────────────┼────────────┘
                         │
                    技术复杂度
```

#### 维度1: 按业务场景分类

| 场景 | 技能集合 | 核心能力 |
|------|---------|---------|
| **获客与投放** | Uplift Modeling, Causal Forest, Doubly Robust | 广告归因、渠道优化、促销决策 |
| **用户运营** | LTV Prediction, Churn Prediction | 价值分层、流失预警、会员体系 |
| **商品与供应链** | Time Series Forecasting, Multi-Echelon Inventory | 需求预测、库存优化、选品决策 |
| **产品优化** | Multi-Armed Bandit, Matrix Factorization | 实验设计、推荐排序、冷启动 |
| **用户体验** | VOC分析, BERT-MoE情感分析 | 舆情监控、产品改进、服务优化 |

#### 维度2: 按技术复杂度分层

```
Level 1: 基础技能 (Foundation)
├── 基础统计推断
├── Python数据分析
└── 机器学习基础

Level 2: 核心技能 (Core) 
├── Uplift Modeling
├── Time Series Forecasting
├── Customer Churn Prediction
└── Multi-Armed Bandit

Level 3: 进阶技能 (Advanced)
├── Causal Forest
├── Doubly Robust Estimation
├── LTV Prediction
└── Matrix Factorization

Level 4: 专家技能 (Expert)
├── Multi-Study Causal Forest
├── HT-GNN (规划中)
└── 动态LTV预测 (规划中)
```

#### 维度3: 按技能关系类型

```
前置依赖链 (Prerequisite Chain)
基础统计推断 → 倾向评分分析 → Uplift Modeling → Causal Forest
                                     ↓
                              Doubly Robust Estimation
                                     ↓
                                   DML

延伸发展链 (Extension Chain)
Uplift Modeling → Causal Forest → MCF
                ↓
         Doubly Robust Estimation → 长期因果推断

协同组合链 (Combination Chain)
Causal Forest + LTV Prediction = 高价值用户精准归因
VOC分析 + Churn Prediction = 情感驱动的流失预警
时序预测 + 库存优化 = 智能补货决策
```

## 四、分类体系实践指南

### 4.1 如何为新技能定位

#### 决策树

```
新技能: X算法
    │
    ├── 1. 确定主领域
    │   ├── 处理因果效应? → 01-因果推断
    │   ├── 处理实验设计? → 02-A/B实验
    │   ├── 处理时序数据? → 03-时间序列
    │   ├── 处理库存物流? → 04-供应链
    │   ├── 处理推荐排序? → 05-推荐系统
    │   ├── 处理用户价值? → 06-增长模型
    │   └── 处理文本数据? → 07-NLP-VOC
    │
    ├── 2. 确定复杂度层级
    │   ├── 依赖≥2个前置技能 → Level 3 (Advanced)
    │   ├── 被≥2个技能依赖 → Level 2 (Core)
    │   └── 无前置/无延伸 → Level 1 或 Level 4
    │
    ├── 3. 确定技能关系
    │   ├── 需要哪些前置技能? → 列出依赖
    │   ├── 可以延伸哪些技能? → 列出发展路径
    │   └── 可以和谁组合? → 列出协同技能
    │
    └── 4. 验证缺口匹配
        ├── 是否填补已有缺口? → 是=高优先级
        └── 是否创造新缺口? → 是=需规划延伸
```

#### 示例: Causal Forest 定位过程

```
1. 领域: 处理因果效应 → 01-因果推断
2. 复杂度: 依赖 Uplift Modeling → Level 3 (Advanced)
3. 关系:
   - 前置: Uplift Modeling, 随机森林
   - 延伸: MCF, 变量重要性分析
   - 组合: Doubly Robust, LTV Prediction
4. 缺口匹配: 填补 Uplift Modeling 延伸缺口 → P0 优先级
```

### 4.2 技能关联编写规范

#### 前置技能选择原则

| 原则 | 说明 | 示例 |
|------|------|------|
| 必要性 | 必须掌握才能理解本技能 | Causal Forest → 需要 Uplift |
| 充分性 | 掌握后学习本技能无障碍 | 有 Uplift 基础即可学 Causal Forest |
| 邻近性 | 概念距离不超过2跳 | Uplift (1跳) → Causal Forest |

#### 延伸技能规划原则

```
延伸方向1: 深度延伸 (技术深化)
Uplift Modeling → Causal Forest → MCF
           ↓
      Doubly Robust

延伸方向2: 广度延伸 (场景扩展)
Uplift Modeling → 动态定价应用
           ↓
      推荐去偏应用

延伸方向3: 组合延伸 (跨界融合)
Uplift Modeling + LTV → 高价值用户归因
```

#### 可组合技能匹配原则

```
互补型组合: 能力互补，1+1>2
├── Causal Forest + LTV = 精准归因+价值分层
├── VOC + Churn = 情感驱动的流失预警
└── 时序 + 库存 = 预测驱动的补货

协同型组合: 流程衔接，端到端
├── Uplift → LTV = 归因到运营闭环
├── 实验 → 推荐 = 快速迭代优化
└── 预测 → 库存 = 需求到供应闭环
```

## 五、图谱维护与演化

### 5.1 定期审查清单

#### 每月审查

- [ ] 新技能是否正确分类到领域
- [ ] 技能关系是否有遗漏或错误
- [ ] 是否有孤岛技能需要建立连接
- [ ] 缺口填补进度是否符合预期

#### 每季度审查

- [ ] 领域分布是否均衡
- [ ] 复杂度层级是否合理
- [ ] 社区结构是否健康
- [ ] 是否需要引入新领域

#### 每年审查

- [ ] 技能是否仍然符合业务需求
- [ ] 技术是否过时需要更新
- [ ] 整体架构是否需要重构

### 5.2 图谱扩展策略

#### 策略1: 缺口驱动扩展

```
发现缺口 → 搜索论文 → 萃取技能 → 填补缺口 → 验证连接

示例:
发现: Causal Forest 无延伸
搜索: Multi-Study Causal Forest
萃取: MCF Skill
填补: 建立 Causal Forest → MCF 连接
验证: 运行图谱分析确认
```

#### 策略2: 业务需求驱动扩展

```
业务需求 → 识别能力缺口 → 映射技术方案 → 萃取/组合技能

示例:
需求: 预测新品上市成功率
缺口: 缺乏上新时机预测能力
方案: Doubly Robust + 时序预测
萃取: 上新时机预测 Skill
组合: 建立与现有技能连接
```

#### 策略3: 技术前沿驱动扩展

```
技术趋势 → 评估业务适用性 → 原型验证 → 萃取技能

示例:
趋势: LLM for Causal Inference
评估: 适合母婴出海客服场景
验证: GPT-4 因果推理实验
萃取: LLM-Causal Skill
```

## 六、可视化图谱规范

### 6.1 图谱绘制标准

#### 节点样式

```
形状: 圆形 (基础) / 方形 (进阶) / 菱形 (专家)
大小: 与入度成正比
颜色: 按领域着色
  - 因果推断: 蓝色
  - A/B实验: 绿色
  - 时间序列: 橙色
  - 供应链: 紫色
  - 推荐系统: 红色
  - 增长模型: 青色
  - NLP-VOC: 粉色

标签: 技能名称 (简洁)
边框: 虚线=规划中, 实线=已完成
```

#### 边样式

```
前置依赖: → (实线箭头)
延伸发展: → (虚线箭头)
可组合: ↔ (双向箭头)

粗细: 与权重成正比
颜色: 与目标节点同色
标签: 关系类型 (可选)
```

### 附录B: 关系强度量化标准

```
权重1.0: 强前置 (必须掌握)
  例: 基础统计 → Uplift Modeling

权重0.8: 强延伸 (自然发展)
  例: Uplift → Causal Forest

权重0.6: 组合推荐 (协同增效)
  例: Causal Forest ↔ LTV

权重0.4: 弱关联 (参考可选)
  例: Time Series → Doubly Robust

权重0.2: 潜在关联 (待探索)
  例: NLP → Causal Inference
```
