# paper-审核

对生成的 Skill 卡片进行质量审核的技能。

## 概述

检查 Skill 卡片的质量，确保符合 Master Prompt 的质量要求。

## 触发方式

用户提及以下内容时触发：
- "审核 skill 卡片" 或 "审核 skill"
- "检查 skill 质量" 或 "质量审核"
- "review skill 卡片" 或 "review skill"
- "检查 skill 卡片"

## 审核标准

### 评分计算方法

总分 = Σ(维度分 × 权重)

| 维度 | 权重 |
|------|------|
| 算法原理 | 25% |
| 应用案例 | 25% |
| 代码模板 | 25% |
| 技能关联 | 10% |
| 商业价值 | 15% |

**通过标准**：总分 ≥ 7分 且 各维度 ≥ 最低要求

### 1. 算法原理 (1-10分)

| 检查项 | 评分标准 |
|--------|----------|
| 非复制重述 | 1=完全复制论文摘要, 10=完全用自己的话重述 |
| 数学直觉 | 1=无公式或解释, 10=公式+直观解释清晰易懂 |
| 关键假设 | 1=未说明, 10=假设和使用条件明确说明 |
| 通俗易懂 | 1=业务人员完全不懂, 10=业务人员能理解核心 |

**维度权重**：25%
**最低要求**：≥6分

### 2. 应用案例 (1-10分)

| 检查项 | 评分标准 |
|--------|----------|
| 场景相关性 | 1=与母婴出海无关, 10=强相关且具体 |
| 业务问题明确性 | 1=模糊不清, 10=问题具体明确 |
| 数据要求清晰度 | 1=未说明, 10=数据要求清晰可获取 |
| 预期产出量化 | 1=无量化指标, 10=产出有具体量化指标 |
| 业务价值量化 | 1=无依据, 10=价值有量化依据 |

**维度权重**：25%
**最低要求**：≥6分

### 3. 代码模板 (1-10分)

| 检查项 | 评分标准 |
|--------|----------|
| 代码完整性 | 1=有占位符不可运行, 10=完整可运行无占位符 |
| 测试用例 | 1=无测试用例, 10=有完整测试用例 |
| 输入输出定义 | 1=未定义, 10=输入输出清晰定义 |
| 代码风格 | 1=风格混乱, 10=风格一致符合规范 |
| 注释说明 | 1=无注释, 10=关键部分有清晰注释 |

**维度权重**：25%
**最低要求**：≥7分（代码质量要求更高）

### 4. 技能关联 (1-10分)

| 检查项 | 评分标准 |
|--------|----------|
| 关联数量 | 1=无关联, 10=关联≥3个Skill |
| 关联逻辑 | 1=无逻辑依据, 10=关联有清晰逻辑 |
| 前后置合理性 | 1=不合理, 10=前后置关系合理 |

**维度权重**：10%
**最低要求**：≥6分

### 5. 商业价值 (1-10分)

| 检查项 | 评分标准 |
|--------|----------|
| ROI预估依据 | 1=无依据, 10=ROI有量化计算依据 |
| 评分说明 | 1=无说明, 10=评分有具体说明 |
| 客观合理性 | 1=明显不合理, 10=评估客观合理 |

**维度权重**：15%
**最低要求**：≥6分

## 审核流程

### Step 1: 读取 Skill 卡片

读取待审核的 Skill 卡片文件。如果未提供具体文件路径，列出可审核的 skill 卡片供用户选择。

### Step 2: 对照检查

根据上述审核标准逐项检查，给出1-10分评分。

### Step 3: 计算总分

总分 = Σ(维度分 × 权重)

检查是否满足通过标准：总分 ≥ 7分 且 各维度 ≥ 最低要求

### Step 4: 标记问题

记录发现的问题：

```
问题列表：
1. [模块]: [问题描述]
2. [模块]: [问题描述]
```

### Step 5: 给出修改建议

针对每个问题，给出具体的修改建议。

### Step 6: 输出审核报告

```markdown
## 审核报告

### 总体评分
- **总分**: X.X/10
- **算法原理**: X/10 (权重25%)
- **应用案例**: X/10 (权重25%)
- **代码模板**: X/10 (权重25%)
- **技能关联**: X/10 (权重10%)
- **商业价值**: X/10 (权重15%)

### 总体评价
[通过/需要修改]

### 问题列表
1. ...
2. ...

### 修改建议
1. ...
2. ...
```

### Step 7: 记录问题到问题库（可选）

如果发现常见问题模式，记录到 `/Users/pray/project/paper_to_skills/paper2skills-vault/07-资源库/审核问题库.md`，供后续审核参考。

## 注意事项

- 严格按照 Master Prompt 的质量要求审核
- 重点关注业务价值是否量化
- 检查代码是否真正可运行
- 确保案例与母婴出海场景相关
