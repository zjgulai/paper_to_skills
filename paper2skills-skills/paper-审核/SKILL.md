# paper-审核

对生成的 Skill 卡片进行质量审核的技能。

## 概述

检查 Skill 卡片的质量，确保符合 Master Prompt 的质量要求。

## 触发方式

用户提及以下内容时触发：
- "审核 skill"
- "检查质量"
- "质量审核"
- "review skill"
- "检查 skill 卡片"

## 审核标准

### 1. 算法原理

检查项：
- [ ] 是否用自己的话重述（非复制论文摘要）
- [ ] 是否包含数学直觉（公式+直观解释）
- [ ] 是否说明关键假设和使用条件
- [ ] 解释是否通俗易懂（业务人员能理解）

### 2. 应用案例

检查项：
- [ ] 案例是否与母婴出海场景强相关
- [ ] 业务问题是否具体明确
- [ ] 数据要求是否清晰
- [ ] 预期产出是否有量化指标
- [ ] 业务价值是否有量化依据

### 3. 代码模板

检查项：
- [ ] 代码是否完整可运行（无占位符）
- [ ] 是否有测试用例
- [ ] 是否有输入输出定义
- [ ] 代码风格是否一致
- [ ] 是否有注释说明

### 4. 技能关联

检查项：
- [ ] 是否关联至少 2 个已有 Skill
- [ ] 关联是否有逻辑依据
- [ ] 前后置技能是否合理

### 5. 商业价值

检查项：
- [ ] ROI 预估是否有量化依据
- [ ] 评分是否有具体说明
- [ ] 评估是否客观合理

## 审核流程

### Step 1: 读取 Skill 卡片

读取待审核的 Skill 卡片文件。

### Step 2: 对照检查

根据上述审核标准逐项检查。

### Step 3: 标记问题

记录发现的问题：

```
问题列表：
1. [模块]: [问题描述]
2. [模块]: [问题描述]
```

### Step 4: 给出修改建议

针对每个问题，给出具体的修改建议。

### Step 5: 输出审核报告

```markdown
## 审核报告

### 总体评价
[通过/需要修改]

### 问题列表
1. ...
2. ...

### 修改建议
1. ...
2. ...
```

## 注意事项

- 严格按照 Master Prompt 的质量要求审核
- 重点关注业务价值是否量化
- 检查代码是否真正可运行
- 确保案例与母婴出海场景相关
